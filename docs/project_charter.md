## Project Title
PyQuiz: Software Testing Project for a Console-Based Quiz Application

## Background
The group project focuses on software testing in a small Python application. The chosen system is a console-based quiz program where the user answers multiple-choice questions and receives a final score. The project is designed to demonstrate unit testing, mocking, dependency injection, Git usage, and structured test documentation.

## Purpose
The purpose of the project is to apply software testing methods in practice and show how requirements, source code, unit tests, and documentation can be connected in a small but structured software project.

## Goals
- Build and structure a Python quiz application with clear modules
- Use dependency injection where suitable
- create unit tests for important parts of the system
- use mocking to isolate dependencies during testing
- document user stories, test strategy, test plan, test cases, and test results
- manage the project in GitHub with commits, branches, and pull requests

## Scope
The project includes:
- a console-based quiz application
- question handling
- answer validation
- score handling
- randomized question order
- unit testing and mocking
- testing documentation

The project does not include:
- graphical user interface
- database integration
- network features
- advanced user account handling

## Success Criteria
The project is considered successful if:
- the application runs from the console
- unit tests pass
- mocking is demonstrated in tests
- the repository structure is clear
- project documentation is complete and aligned with the code
